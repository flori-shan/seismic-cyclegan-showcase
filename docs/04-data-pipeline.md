# 04 · 数据管道与预处理

## 1. 数据来源

### 1.1 GOM 2D 公开数据集

- 来源：Madagascar 生态，`sfpatch` 等工具处理  
- 特点：二维炮集，适合算法验证  
- 处理：重叠 patch 切分，训练 / 测试分离

### 1.2 自研山地数据集

- 公司内部采集数据（**不可公开**）  
- 尺寸：200×200 或 1024×1024 patch  
- 场景：山地复杂地质，盐丘相关多次波

### 1.3 缺失道构造

模拟真实采集缺失：

```
完整炮集 → 随机/规则删除 30% 检波道 → 域 X
完整炮集本身              → 域 Y
```

**近偏移实验**：额外删除**前 5 道**，模拟近距检波器缺失。

## 2. 预处理流程

```mermaid
flowchart LR
    A[原始炮集 SEG-Y] --> B[归一化 / 振幅对齐]
    B --> C[裁剪 ROI]
    C --> D[overlap 切块]
    D --> E[域 X / 域 Y 配对加载]
    E --> F[tf.data 批处理]
```

### 2.1 振幅归一化（关键）

**问题**：域 X（缺失道）与域 Y（完整道）灰度范围不一致 → 生成图**颜色过深/过浅**。

**处理**：

- 两域统一 min-max 或 z-score 归一化  
- 训练前检查直方图分布  
- 推理后反归一化用于波形显示

### 2.2 切块策略

| 参数 | 取值 | 说明 |
|------|------|------|
| patch 尺寸 | 200×200 / 1024×1024 | 小 patch 训练快；大 patch 保留更多同相轴 |
| overlap | 有 / 无 | **有 overlap** 显著改善边缘与拼接 |
| 训练/测试 | 目录分离 | 避免泄漏 |

### 2.3 Madagascar 参考命令（GOM）

```bash
# 概念性示例 — 实际参数以项目配置为准
sfpatch < input.rsf > patches.rsf overlap=...
```

## 3. TensorFlow 数据加载

### 3.1 两种实现路径

1. **`tensorflow_datasets`**  
   - 自定义 `GeneratorBasedBuilder`  
   - 注册 `train` / `test` split  
   - 适合标准化实验记录

2. **`tf.data` 原生管道**  
   - `from_generator` 或 `list_files`  
   - `map` 解码、`batch`、`prefetch`  
   - 灵活调试

### 3.2 注意事项

| 问题 | 处理 |
|------|------|
| 测试集 shape 不一致 | 统一 resize 或按尺寸分桶 batch |
| 预测时乱序 | **关闭 shuffle**，按文件名排序 |
| 内存 | `prefetch(AUTOTUNE)`，控制 batch size |

## 4. 数据增强（可选）

- 水平翻转（谨慎：道方向有物理意义）  
- 小幅振幅缩放  
- **未采用**剧烈几何变换，避免破坏同相轴物理意义
